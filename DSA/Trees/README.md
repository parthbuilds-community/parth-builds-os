# Trees

Trees are hierarchical data structures consisting of nodes connected by edges. Unlike linear data structures such as arrays and linked lists, trees organize data in a parent-child relationship.

## Key Terminologies

* **Root Node**: The topmost node of the tree.
* **Parent Node**: A node that has one or more child nodes.
* **Child Node**: A node that descends from another node.
* **Leaf Node**: A node with no children.
* **Height of Tree**: The length of the longest path from the root to a leaf.
* **Depth of Node**: The number of edges from the root to that node.

## Types of Trees

* Binary Tree
* Binary Search Tree (BST)
* AVL Tree
* Segment Tree
* Trie
* Heap

## Common Tree Traversals

1. Inorder Traversal (Left → Root → Right)
2. Preorder Traversal (Root → Left → Right)
3. Postorder Traversal (Left → Right → Root)
4. Level Order Traversal (BFS)

## Applications of Trees

* File System Organization
* Database Indexing
* Search Operations
* Routing Algorithms
* Expression Evaluation
* Hierarchical Data Representation

## Time Complexity of Common BST Operations

| Operation | Average Case | Worst Case |
| --------- | ------------ | ---------- |
| Search    | O(log n)     | O(n)       |
| Insert    | O(log n)     | O(n)       |
| Delete    | O(log n)     | O(n)       |
