# How to Actually Learn Coding (Not Just Collect Tutorials)

## Logic Over Syntax
- Programming is problem-solving, not memorizing commands
- Break problems: "What steps would I take manually?"
- Syntax is searchable—logic isn't
- Practice: Explain your solution in plain English first

## Read Code Before Writing
- Find GitHub repos in your learning area
- Trace through code line by line
- Notice patterns: how do they organize files? name variables?
- Steal structure, not verbatim code

## Debugging is Half the Job
- **Print/log everything** when something breaks
- Read error messages top to bottom
- Google the error with your language name
- Comment out code to isolate the problem
- Use a debugger (built into most IDEs)

## Projects Over Courses
- Courses teach concepts, projects teach skills
- Start projects before you feel "ready"
- Finish one small project before starting another
- Document what you learned in each project

## Action Steps
1. Pick a project idea today
2. Build the simplest version possible
3. Break when you hit a wall (you will)
4. Google, debug, repeat
5. Ship it even if it's ugly
