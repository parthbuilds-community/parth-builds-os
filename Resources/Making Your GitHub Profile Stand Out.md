# Making Your GitHub Profile Stand Out

## Profile README
- Create a README.md in a repo named after your username
- Include: brief intro, tech stack, current focus
- Add badges for languages/tools (shields.io)
- Keep it under 100 words

## Pinned Projects (Quality > Quantity)
- Choose 3-4 projects that work and solve real problems
- Each needs a detailed README with:
  - What it does and why you built it
  - Technologies used
  - Setup instructions
  - Screenshots or live demo link

## Contribution Graph
- Commit regularly, even small changes
- Recruiters look for consistency
- Use private repos if you must, but make some work public
- Contributions to open source count too

## Professional Commits
- Bad: "fixed stuff", "update", "asdf"
- Good: "Add user authentication", "Fix navbar bug on mobile"
- Format: "Verb + what changed"
- Shows attention to detail

## Profile Checklist
- [ ] Profile picture (professional or avatar)
- [ ] Bio with your role/interests
- [ ] Location and website/portfolio
- [ ] Pinned repos with READMEs
- [ ] Recent activity (commits in last month)
